# 02 - هيكلة المشروع (Project Structure)

تم تنظيم المشروع ليكون قابلاً للصيانة (Maintainable) وقابلاً للتوسع (Scalable) باتباع مبادئ الـ Clean Code وفصل المسؤوليات.

## المجلدات الرئيسية

### 1. `Data/`
يحتوي على `DbContext` (سياق قاعدة البيانات) الذي يمثل الطبقة التي تتواصل مع قاعدة البيانات مباشرة.
- الملف الأساسي: `ClinicDbContext.cs`

### 2. `Models/`
يحتوي على الكيانات (Entities) التي تمثل جداول قاعدة البيانات (مثل `Patient`, `Appointment`).

### 3. `Extensions/`
تم استخدامه لتبسيط ملف `Program.cs`. بدلاً من كتابة كل الإعدادات في مكان واحد، قمنا بإنشاء دوال توسعة (Extension Methods) مثل:
- `ApplicationServiceExtensions.cs`: لحقن الاعتماديات (Dependency Injection) وإعداد قاعدة البيانات.
- `IdentityServiceExtensions.cs`: لإعداد نظام الهوية والمستخدمين.

### 4. `Services/`
يحتوي على منطق العمل (Business Logic). الخدمات تقوم بمعالجة البيانات قبل حفظها أو إرجاعها للمستخدم.

### 5. `Middleware/` (البرمجيات الوسيطة)
يحتوي على معالجات مخصصة تمر خلالها الطلبات (Requests)، مثل معالجة الأخطاء (Global Exception Handling) لتوفير ردود موحدة عند حدوث مشاكل.

## ملف `Program.cs`
أصبح الملف بسيطاً جداً وواضحاً بفضل استخدام الـ Extensions. هو مجرد نقطة دخول تقوم باستدعاء دوال الإعداد وتشغيل التطبيق.
